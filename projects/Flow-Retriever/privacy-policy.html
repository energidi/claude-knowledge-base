Privacy Policy

Flow Retriever Chrome Extension — Last updated: April 24, 2026

Overview

Flow Retriever is a Chrome Extension that allows Salesforce users to extract Flow metadata as JSON directly from the browser. This policy explains what data the extension accesses, how it is used, and what is never collected.

Data We Access

The extension reads one piece of data from your browser:

  • Salesforce Session Cookie (sid): The extension reads your active Salesforce session token from the browser cookie store. This token is used solely to authenticate a single API request to your Salesforce org's Tooling API, which retrieves the Flow metadata you requested.

How Your Data Is Used

  • The session token is used in memory, for the duration of a single API call, and is immediately discarded.
  • It is sent only to your own Salesforce org (the same domain you are already logged in to).
  • The Flow JSON returned by the API is passed through the extension's internal message bus to the active browser tab, where it is either copied to your clipboard or downloaded to your local computer — whichever you choose. This transit is local to your browser and is not transmitted to any external server.

Data We Do Not Collect

  • We do not store any data on external servers.
  • We do not transmit any data to third parties.
  • We do not log, track, or retain your session token, Salesforce data, or any personal information.
  • We do not use analytics or tracking tools of any kind.
  • We have no backend infrastructure. All processing occurs locally in your browser.

Local Storage

The extension uses Chrome's storage.sync API to save your preferences (for example, whether you prefer to copy JSON to clipboard or download it as a file). This preference is synced across your Chrome profile via your Google account. No Salesforce data or session information is ever written to storage.

Permissions Used

  • cookies: To read the Salesforce session token for API authentication.
  • storage: To persist your extension preferences across devices via Chrome sync.
  • alarms: To keep the extension's background service worker alive during API requests (a Chrome MV3 requirement).
  • Host permissions (*.salesforce.com, *.lightning.force.com, *.force.com): To inject the retrieval button into Salesforce Flow Builder pages and to call the Salesforce Tooling API.
